## 流行的AI智能体

### 开源AI Agent框架分类

| 类别 | 典型代表框架 |
| :--- | :--- |
| **工作流编排** | LangGraph |
| **多智能体协作** | CrewAI, MetaGPT, AutoGen |
| **可视化低代码平台** | Dify |
| **自主进化/自优化** | EvoMap, EvoAgentX |
| **全能模块化开发框架** | LangChain |
| **轻量极简类** | Hermes Agent |


### 流派一：工作流编排类 (Workflow Orchestration)

这个流派把智能体的思考过程看作一系列有序的节点，你可以像画流程图一样设计智能体的行为。

- **核心特征**：强调**状态管理**，能处理复杂的多轮对话和条件分支，流程清晰可控，适合需要严谨逻辑的业务。
- **典型框架（LangGraph）**：这个流派的开创者与集大成者。它基于**有向图 (DAG)** 和**状态机**的理念，支持循环和条件路由，能将复杂的智能体行为拆解为可组合的节点。它适合构建如金融、医疗等需要审计追踪和强逻辑控制的场景。

### 流派二：多智能体协作类 (Multi-Agent Collaboration)

这个流派模拟人类团队协作，通过定义不同角色的专业Agent（一个当产品经理，一个当码农）来分工处理复杂任务。

- **核心特征**：核心是**角色扮演**和**任务分工**，多个Agent各司其职，通过协作完成单一Agent难以搞定的目标。
- **典型框架**：
    - **CrewAI**：以轻量和逻辑直观著称，就像搭建一个AI“剧组”或“项目组”，可以快速定义“研究员”、“写手”、“审核员”等角色，让他们协同完成任务。它的优势是上手快，10分钟就能搭出一个多智能体协作原型。
    - **MetaGPT**：是CrewAI的更专业版本，它模拟一个标准的软件公司，内置了产品经理、架构师、工程师等完整SOP流程，只需一句话需求，就能输出用户故事、API设计、代码乃至可运行的原型。

### 流派三：可视化低代码平台 (Visual Low-Code)

这个流派面向“快”字当头的场景，通过拖拽式界面和预置组件，以极低成本快速构建和验证AI应用想法。

- **核心特征**：**零/低代码**、**可视化拖拽**、**开箱即用**的组件库，极大降低了开发门槛。
- **典型框架（Dify）**：这个流派的优秀代表，是个可视化的LLMOps平台。它提供了拖拽式的**工作流**和**聊天流**编排界面，集成了丰富的预置组件和插件，让你能快速搭建知识库问答、智能客服等应用。

### 流派四：自主进化/自优化类 (Self-Evolution)

这是比较前沿的流派，目标是让框架或Agent自己学会“进化”，自动找到或优化出更好的工作流和配置。

- **核心特征**：工作流的**自动生成、执行、评估和进化**，让系统“越用越聪明”。
- **典型框架**：
    - **EvoMap**：定位为“AI Agent自我进化的基础设施”，通过基因进化协议（GEP），实现**Agent之间的经验共享、继承与协作**。
    - **EvoAgentX**：是一个自动化研究框架。你给出任务目标，它能**自动完成工作流构建、配置、执行与持续优化**，集成了多种进化算法来迭代改进。

### 其他流派与“全能选手”

- **流派五：全能型/模块化开发框架**：这类框架功能最全，像一个大而全的军火库，你几乎可以用它构建所有类型的Agent。**典型项目是LangChain**，拥有庞大的模块生态系统，从RAG到工具调用无所不包。
- **流派六：轻量极简类**：这类框架聚焦单体Agent（单兵作战），部署轻量，强调自主性和记忆能力。**典型项目是Hermes Agent**，能在你自己的服务器上运行，具备跨会话记忆和自学习能力，适合个人助手、自动化写作等场景。

### 主要框架速查对比表

为了让你快速选型，这里是这些框架的核心信息对比：

| 框架 (Framework) | 流派 | 一句话简易定位 | 适用场景 (开发者视角) |
| :--- | :--- | :--- | :--- |
| **LangGraph** | 工作流编排 | 有状态图工作流引擎，流程控制力强 | 金融/医疗复杂业务流、多轮对话系统 |
| **CrewAI** | 多智能体协作 | 角色扮演式AI团队，上手快，逻辑直观 | 内容创作、市场调研、自动化报告生成 |
| **MetaGPT** | 多智能体协作 | 模拟软件公司的AI团队，流程化要求高 | 自动化软件开发、需求分析、原型生成 |
| **Dify** | 可视化低代码 | LLMOps平台，拖拽式搭建AI应用 | 企业内部工具、客服机器人、快速原型验证 |
| **AutoGen** | 多智能体协作 | 微软出品，支持人机协同和代码执行 | 自动化代码生成与调试、科研实验 |
| **LangChain** | 全能模块化 | 通用模块库，生态庞大，集成功能极多 | 需要高度自定义和复杂集成的项目 |
| **EvoAgentX** | 自主进化 | 工作流自动化生成与进化优化 | 探索前沿、需要自适应优化的研究型项目 |
| **EvoMap** | 自主进化 | Agent间经验共享与能力继承平台 | 构建去中心化、协同进化的Agent生态 |
| **Hermes Agent** | 轻量极简 | 轻量、自托管的自主单体智能体 | 个人AI助手、自动化脚本、私有化部署 |

看到你之前也问过模型和生态的问题，针对你“实践派、行动派”的需求，我给两个最直接的选型切入点：

1.  **如果你是“想快速跑通业务闭环”的开发/创业者**：建议先看**Dify**。它能让你在一小时内搭出可用的AI应用原型，优先验证想法，避免前期陷入技术细节。
2.  **如果你是“想构建复杂、强逻辑AI系统”的工程师**：可以从**LangGraph**入手。它虽然学习曲线陡峭，但带来的状态控制、流程追溯和调试能力，是构建生产级复杂Agent系统的关键。

> **补充说明**：除了提到的**EvoMap**和**EvoAgentX**，**Microsoft AutoGen**在很多场景也被归类为多智能体协作框架，有时会与CrewAI、MetaGPT并称“多Agent三剑客”。
